## Prerequisites

1. Backend running with Docker and PostgreSQL:
   ```powershell
   cd backend
   $env:DATABASE_URL = "postgresql+asyncpg://postgres:postgres@localhost:5432/agent_b"
   python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
   ```

2. Frontend running:
   ```powershell
   cd frontend
   npm run dev
   ```

3. Access the app at http://localhost:3000

## Test 1: Backend Offline State

**Purpose:** Verify the app handles backend unavailability gracefully.

**Steps:**
1. Stop the backend server (Ctrl+C in terminal)
2. Refresh the frontend at http://localhost:3000
3. Observe the UI

**Expected Results:**
- [ ] "Cannot connect to backend" message is displayed
- [ ] A "Retry connection" button is visible

**Recovery:**

2. Click "Retry connection"

## Known Limitations

1. **SSE token in query string** - Token visible in server logs. Acceptable for local-first tool.
2. **localStorage XSS exposure** - Token in localStorage is vulnerable to XSS. Mitigated by short token lifetime.
3. **No token refresh** - User must re-login when token expires (24h default).
4. **Provider list hardcoded** - Must update `LLM_PROVIDERS` constant when B2.0 adds new providers.
