# B1.5 Session Management UI - Manual Test Script

## Prerequisites

1. Docker is running (for PostgreSQL database)
2. Backend is running on port 8000
3. Frontend is running on port 3000

## Setup

```powershell
# Terminal 1 - Start backend
cd backend
$env:DATABASE_URL = "postgresql+asyncpg://postgres:postgres@localhost:5432/agent_b"
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

# Terminal 2 - Start frontend
cd frontend
npm run dev
```

## Test Cases

### TC1: Session Sidebar Displays
- [ ] Open http://localhost:3000
- [ ] Verify session sidebar is visible on the left side
- [ ] Verify "Sessions" header with plus icon is visible
- [ ] Verify sidebar width is approximately 256px

### TC2: Empty State
- [ ] If no sessions exist, verify "No sessions yet" message is displayed
- [ ] Verify the message includes "Create one to get started"

### TC3: Create Session
- [ ] Click the "+" button in the Sessions header
- [ ] Verify a new session "New Session" appears in the list
- [ ] Verify the new session is automatically selected (highlighted)
- [ ] Verify the ReasoningPanel is ready to accept messages

### TC4: Switch Sessions
- [ ] Create at least 2 sessions
- [ ] Click on a non-active session
- [ ] Verify the clicked session becomes highlighted (blue border)
- [ ] Verify the previous session loses highlight
- [ ] Verify messages in ReasoningPanel update to the selected session

### TC5: Rename Session
- [ ] Hover over a session in the list
- [ ] Verify pencil (edit) icon appears
- [ ] Click the pencil icon
- [ ] Verify inline edit mode activates (input field appears)
- [ ] Type a new name and press Enter
- [ ] Verify the session title updates
- [ ] Test pressing Escape cancels the edit

### TC6: Delete Session (Non-Current)
- [ ] Hover over a session that is NOT currently selected
- [ ] Click the trash icon
- [ ] Verify delete confirmation dialog appears
- [ ] Verify session title is shown in dialog
- [ ] Verify "This will permanently delete..." warning text
- [ ] Click Cancel - verify dialog closes and session remains
- [ ] Click trash icon again, then click Delete
- [ ] Verify session is removed from list

### TC7: Delete Current Session
- [ ] Select a session (make it current)
- [ ] Delete the current session
- [ ] Verify after deletion:
  - Session is removed from list
  - No session is selected (no highlight)
  - ReasoningPanel shows empty/welcome state

### TC8: Delete Last Session
- [ ] Delete all sessions until one remains
- [ ] Delete the last session
- [ ] Verify empty state "No sessions yet" appears

### TC9: Session Timestamps
- [ ] Create a new session
- [ ] Verify relative timestamp shows (e.g., "just now")
- [ ] Send a message in the session
- [ ] Verify timestamp updates (e.g., "1m ago")

### TC10: Error Handling
- [ ] Stop the backend server
- [ ] Refresh the page
- [ ] Verify error message "Failed to load sessions" appears
- [ ] Verify "Try again" retry button is visible
- [ ] Start the backend server
- [ ] Click "Try again"
- [ ] Verify sessions load successfully

### TC11: Loading State
- [ ] Refresh the page
- [ ] Verify loading spinner appears briefly while sessions load
- [ ] Verify spinner disappears when sessions are loaded

## API Verification (PowerShell)

```powershell
# List sessions
Invoke-WebRequest -Uri http://localhost:8000/api/v1/sessions |
  Select-Object -ExpandProperty Content

# Create session
Invoke-WebRequest -Uri http://localhost:8000/api/v1/sessions -Method POST `
  -Headers @{"Content-Type"="application/json"} `
  -Body '{"title": "API Test Session"}' |
  Select-Object -ExpandProperty Content

# Rename session (replace SESSION_ID)
Invoke-WebRequest -Uri "http://localhost:8000/api/v1/sessions/SESSION_ID" -Method PATCH `
  -Headers @{"Content-Type"="application/json"} `
  -Body '{"title": "Renamed Session"}' |
  Select-Object -ExpandProperty Content

# Delete session (replace SESSION_ID)
Invoke-WebRequest -Uri "http://localhost:8000/api/v1/sessions/SESSION_ID" -Method DELETE
```
