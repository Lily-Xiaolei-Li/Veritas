## 🐛 Debug 记录

### 端口冲突问题
**症状：** Frontend 启动失败，报 EACCES 错误

**原因：** 
1. 旧进程没有正确关闭
2. IPv6 和 IPv4 同时监听导致冲突

**解决：**
```powershell
# 同时杀掉 IPv4 和 IPv6 上的进程
netstat -ano | findstr :3000
taskkill /F /PID <pid>
```

### State 管理问题
**症状：** 折叠状态在组件间不同步

**解决：** 使用 Zustand store 统一管理 UI 状态
- 新增 `frontend/src/lib/store/index.ts`
- 所有折叠状态通过 store 管理

## 💡 经验教训

1. **启动脚本要简单** - 用户不应该看到多个启动选项
2. **UI 状态集中管理** - 用 Zustand 而不是 props drilling
3. **折叠设计要一致** - 所有可折叠区域用相同的交互模式
4. **端口管理要小心** - Windows 上 IPv6 是个坑
