### 2. Events 面板显示真实内容 + 实时响应
**问题：** 现在只显示 "Waiting for brain activity..."，什么都没有

**需要显示：**
- ✅ **System Prompt** — 发出去的完整系统提示词（确认 persona 正确）
- ✅ **User Message** — 用户输入
- ✅ **Context** — 附加的上下文（不是全文，而是实际发送的片段）
- ✅ **Brain Activity** — LLM 思考过程（如果 provider 支持 streaming thinking）

**🆕 实时响应需求：**
- 当用户在 Persona 下拉菜单选择不同 persona 时
- Events 面板**立即**更新显示该 persona 的 system prompt
- 不需要等到发送消息，选择即显示
- 这样用户可以确认选对了 persona

**UI 方案：**
```
┌─ Events ─────────────────────────────┐
│ 🎭 Current Persona: The Templator    │  ← 实时更新
│ ─────────────────────────────────────│
│ 📤 System Prompt                     │  ← 选择 persona 立即显示
│   You are The Templator, you help    │
│   structure academic papers...       │
│                                      │
│ 📝 User Message                      │
│   Compare the 2 and tell me...       │
│                                      │
│ 📎 Context (2 chunks)                │
│   [Chwastiak 2013] p.32-43...        │
│   [Radcliffe 2008] p.15-20...        │
│                                      │
│ 🧠 Thinking... (streaming)           │
│   Analyzing the differences...       │
└──────────────────────────────────────┘
```

**实现要点：**
- Persona 选择器触发事件 → 更新 Events 面板状态
- 需要前端存储每个 persona 的 system prompt（或从后端获取）
- 用 Zustand store 管理当前 persona 和对应 prompt

**涉及文件：**
- `frontend/src/components/workbench/ConsolePanel.tsx` (或新建 EventsPanel)
- `frontend/src/components/chat/PersonaSelector.tsx` - 触发更新事件
- `frontend/src/lib/store/index.ts` - 添加 persona prompt 状态
- `backend/` - 可能需要 API 返回 persona 的 system prompt

**复杂度：** ⭐⭐⭐ 较高（需要后端配合 + 状态管理）

### 4. LLM 直接输出 Artifact
**需求：** 如果 prompt 要求返回文件，LLM 可以直接创建 artifact

**例子：**
```
User: "Summarize this paper and save as summary.md"
AI: [直接在 artifacts 里创建 summary.md，而不是在聊天里输出]
```

**实现方案：**
- 在 system prompt 里告诉 LLM 可以用特殊格式输出 artifact：
  ```
  <<<ARTIFACT:filename.md>>>
  content here
  <<<END_ARTIFACT>>>
  ```
- 前端解析这个格式，自动创建 artifact
- 聊天里显示："✅ Created artifact: summary.md"

**涉及文件：**
- `backend/app/agent/` - system prompt 模板
- `frontend/` - 解析 artifact 格式，调用 create artifact API

**复杂度：** ⭐⭐⭐ 较高

### 5. Editor 内 AI 直接编辑
**需求：** 选中文本 → 让 AI 重写 → AI 直接在 artifact 里修改

**交互流程：**
1. 用户在 artifact editor 里选中一段文字
2. 右键或快捷键 → "Ask AI to rewrite"
3. 弹出小输入框："Rewrite in what way?"
4. AI 处理后，直接替换选中的文本

**UI 方案：**
```
[选中文字] → 右键菜单
  ├─ 🤖 Ask AI to rewrite...
  ├─ 📝 Simplify
  ├─ 🎯 Make more academic
  └─ 🔄 Expand
```

**涉及文件：**
- `frontend/src/components/artifacts/ArtifactPreview.tsx` - editor 组件
- 可能需要集成一个支持选区操作的 editor（Monaco / CodeMirror）

**复杂度：** ⭐⭐⭐⭐ 高（涉及 editor API）
